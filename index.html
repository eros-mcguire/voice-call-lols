1

Install VB-Cable (free)

Download and install the free VB-Audio Virtual Cable. This creates a virtual audio device on your system. Run the installer as Administrator and reboot if prompted.

2

Set Browser Output to VB-Cable

In Chrome/Edge: Click the 🔊 icon in the browser tab → Output: CABLE Input (VB-Audio). This routes the browser's audio (your mixed call) into the virtual cable.

3

Set Discord Input to VB-Cable Output

In Discord: Settings → Voice & Video → Input Device → CABLE Output (VB-Audio). Discord will now receive the mixed audio of both call participants as a microphone.

4

Join a Discord Voice Channel

Both voices from your VoiceBridge call will now appear as your microphone input in Discord, OBS, Zoom, or any other app.

Tip: Use Chrome's per-tab audio output control (click the speaker icon in the address bar on any tab) for granular control. VB-Cable Basic is free; VB-Cable A+B adds more virtual devices for $8.
1

Install BlackHole (free)

Download BlackHole 2ch — a free, open-source virtual audio driver for macOS. Install it and restart your browser.

2

Create a Multi-Output Device

Open Audio MIDI Setup (Spotlight → "Audio MIDI Setup") → click + → Create Multi-Output Device → check both BlackHole 2ch and your speakers/headphones.

3

Set Browser Output

In Chrome: click the speaker icon on this tab → select Multi-Output Device (or BlackHole 2ch directly). You'll still hear audio through your headphones while BlackHole captures it.

4

Set Discord Input to BlackHole

Discord Settings → Voice & Video → Input Device → BlackHole 2ch. Both voices are now your Discord mic input.

Tip: If you don't hear audio through headphones, make sure your Multi-Output Device includes your headphones AND BlackHole together.
1

Create a PulseAudio Null Sink

Run in terminal: pactl load-module module-null-sink sink_name=voicebridge sink_properties=device.description=VoiceBridge. This creates a virtual audio device.

2

Route Browser Audio to Null Sink

Open pavucontrol (PulseAudio Volume Control) → Playback tab → find your browser → change its output to VoiceBridge.

3

Set Discord Input to Monitor

In pavucontrol → Recording tab → set Discord's input to Monitor of VoiceBridge. Or in Discord Settings → Voice & Video → Input: Monitor of VoiceBridge.

4

Make it Permanent (optional)

Add the pactl command to ~/.config/pulse/default.pa to persist across reboots. PipeWire users: use pw-loopback instead.

PipeWire users: Run pw-loopback --capture-props='media.class=Audio/Sink' to create a loopback sink. Then follow steps 2–3 using qpwgraph or pavucontrol.
